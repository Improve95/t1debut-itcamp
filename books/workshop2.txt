паттерн душитель - для перехода от легаси с новой архитектуре
anti corruption layer - для создания новой архитектуры без влияния легаси кода

бд:
у каждого микросервиса своя
у всех микросервисов одна бд

бд одна на чтение и одна на запись:
master - slave
бд на чтение должна синхронизироваться с бд на запись

паттерны управления событий:
event sourcing - поиск событий: сохраняет всю историю объекта
нельзя удалить событие
всегда есть состояние, есть возможность отката

шаблон сага - шаблон для отката транзакций

оркестрация, хореография - два подхода реализации паттерна сага, 
где один - централизованный подход,
а второй - каждый микросервис следит за собой сам(нарушает прицин single responcibility),
здесь есть очередь событий, где каждый микросервис знает как откатывать события
